02Typography

Two fonts. Both bilingual. No compromise on Hebrew.

David Libre (display) handles headlines, eyebrows, section markers, and pull quotes. Mid-century Hebrew serif (Ismar David, 1950s) — modern enough to read editorial, distinctly not liturgical. Three weights: 400 / 500 / 700. Rubik (UI) handles everything functional: body, prices, navigation, buttons, micro-text. The pair gives JLM a voice no other Israeli wine site has. No italic anywhere — Hebrew italic leans against right-to-left reading, so the system uses color and weight shifts for emphasis instead.

08Image Treatment

The catalog spec is a feature.

Catalog standard (locked): 1000px square, transparent PNG, bottle visible, year obscured. Boxed wine shows the box. Accessories use the same standard. Renders at 60% of the card photo well — the bottle has room to breathe.

Grounding treatment: a soft radial shelf shadow under the bottle (low-opacity ink ellipse, 8px tall, blurred), inside the photo well. Prevents the floating-sterile look of pure transparent PNGs on white. The shadow stays even when the bottle nudges up on hover — the bottle moves over its shadow, the shadow stays put.

Bundles vs packages: one card chassis. Catalog grouping (separate Bundles / Packages sections) is the primary signal. On the card itself: ink "Bundle" tag pill + cream-base photo well, OR terracotta "Package" tag pill + warm tinted gradient photo well. Bundle imagery never identifies specific products (contents may rotate); package imagery can be richer because contents are fixed.

Discount badges: WooCommerce native sale flash (span.onsale). The Product Bundles plugin sets is_on_sale() through the standard API, so the flash fires automatically on any package or sale-priced product. Theme overrides default styling to amber sticker, top corner. Optional woocommerce_sale_flash filter swaps "Sale!" for "Save ₪50" / "חיסכון ₪50" by reading regular_price - sale_price.

Format pipeline: archival masters stay PNG-1000 (lossless, source of truth). Theme serves WebP-with-alpha at runtime via <picture> + srcset. Reduces a 240 KB PNG to ~60–80 KB WebP with no visible loss. Fallback to PNG for ancient browsers (~0.3% of traffic).

Sizes per slot: card thumb 360w, card 2x retina 720w, single product hero 1000w, single product zoom 2000w (upscaled if needed). All cropped square. Lazy-loaded except for above-the-fold.

Lifestyle / hero / blog imagery: separate spec — atmospheric, food-and-table moments, bright daylight (not vineyard sunsets). Catalog and lifestyle never appear in the same component.

09Production Features

What the new theme inherits — restyled, not rebuilt.

The live site already runs sticky header, header search, age verification, Complianz consent, Mailchimp signup (MC4WP), WhatsApp click-to-chat, the Elementor accessibility widget, WPML, and Product Bundles. The theme replacement keeps these plugins; the design system tells them how to look.

Sticky header: compresses on scroll (logo shrinks, nav stays, cart + search + lang switcher persist). Shadow appears only when scrolled. Background remains --c-base with a 1px --c-ink-12 bottom border.

Header search: always-visible input next to the language switcher. WooCommerce native product search backs it. Field uses the footer-input pattern (transparent, ink border, terracotta on focus).

Mini-cart: opens as a drawer from the cart icon click. Contains the free-shipping monitor at the top, line items in compact horizontal layout, totals + checkout CTA at the bottom. Not always-visible — the header cart icon + count badge does the persistent signaling.

Free-shipping monitor: single component, multiple positions. Slim sticky strip under the header on cart-bearing pages, top of the mini-cart drawer, top of the cart page, above the PDP add-to-cart. Data-driven (current cart total + threshold).

WhatsApp floating icon: bottom-corner circle, opposite horizontal corner from the floating add-to-cart bar. 4-6 second appearance delay. Tap opens chat with +972555174805. Both icons flip together in RTL.

Age verification gate: appears in the language of the page the user landed on. Footer of the modal carries a small language toggle that re-routes to the equivalent page in the other language. "Yes 18+" stores cookie/localStorage, proceeds to original URL. "No" redirects off-site.

Complianz cookie banner: existing plugin. Theme styles the banner to match the system: cream-base background, ink text, terracotta CTA. Close button must not affect scroll position (current bug — investigate or replace).

Newsletter form (MC4WP): styled to match the footer-form pattern. Plugin emits the markup, theme CSS does the rest.

Accessibility widget: Elementor a11y widget remains for Israeli law compliance. Self-contained 38 KiB, theme-agnostic.

Catalog filter UI: contextual — only renders attribute filters that apply to the current result set. Sensory triad, when shown, ordered intensity → complexity → acidity. Bundles/packages have no sensory attributes; dry reds/whites each have 2 of 3. Kashrut and grapes are not stored as attributes — no catalog filter.

Bilingual content: bundle and package products are managed independently in EN and HE — not translations. Design must not assume parity. WPML still drives the language switch and dir attribute for everything else.
