- [x] 2026-05-15 (re-confirmed 2026-05-27): **Admin Projects task delete — claims completion but leaves records.** RESOLVED 2026-05-28 Session C @149 deploy @153. Root cause was a row-index race: frontend `cancelSelectedTasks` fired N parallel `WebAppTasks_deleteTask` calls; backend deleted by row index per call; parallel readers computed stale indices against each other's deletes, producing "Task not found" false positives + wrong-row deletes + display lag. Frontend rewritten to single call with detailed failure alert. Singular `WebAppTasks_deleteTask` kept in place for single-task cancel. Smoke confirmed.
